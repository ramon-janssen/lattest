newtype Expr t Source #

Expr: value expression 1. User can't directly construct Expr (such that invariants will always hold) 2. User can still pattern match on Expr using ExprView 3. Overhead at run-time is zero. See https://wiki.haskell.org/Performance/Data_types#Newtypes
